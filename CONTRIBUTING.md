# Contributing to vens

Thank you for your interest in contributing to vens! We appreciate your help in making contextual vulnerability risk scoring better for everyone.

## How to Contribute

### Reporting Issues

- **Search existing issues** before creating a new one
- Provide a clear description of the problem
- Include steps to reproduce, expected behavior, and actual behavior
- Share relevant configuration files and error messages

### Submitting Pull Requests

1. **Fork the repository** and create a new branch from `main`
2. **Make your changes** following the code style of the project
3. **Test your changes** thoroughly
4. **Commit with clear messages** describing what and why
5. **Submit a PR** with a description of your changes

### Development Setup

```bash
# Clone your fork
git clone https://github.com/YOUR_USERNAME/vens.git
cd vens

# Install dependencies
go mod download

# Build
make binaries

# Run tests
make test
```

## Questions?

Open an issue with your question, and we'll be happy to help!
